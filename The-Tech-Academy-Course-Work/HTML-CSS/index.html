Why did I choose software development?

There's countless reasons why I ultimately decided to take up learning code through Tech Academy and ultimately through the internet by my own determination.

First Being...

Job security. There isn't a better example of a limitless and exponentially growing industry in America and across the globe. There are some routes you can only take with a higher education, however this industry allows people to join based on their self driven attitude and ability to adapt to situations on the fly. In just a week of studying SQL I learned a solid understanding of the syntax and was able to create an entire mock Library inventory, with multiple tables and developed stored functions and queries for this database to allow for specified results without returning gigantic tables.

Here is an example of that code...


/*#7 Retrieve title of Stephen King book, and # of copies owned by Central*/
select count(lb.BranchName) as [BookCount], lb.BranchName, b.Title
from Book as B inner join Book_Authors as BA
	on b.Book_ID = ba.Book_ID
		inner join Book_Loans as BL
	on  b.Book_ID = BL.Book_ID
		inner join Library_Branch as LB
	on BL.BranchID = LB.BranchID
Where Ba.AuthorName = 'Stephen King'
	and lb.BranchName = 'Central'
Group By lb.BranchName, b.Title

/*Stored Procedure for One or more of the above queries*/

Create Procedure SearchBookAuthor @Author varchar(30) = NULL
AS
select count(lb.BranchName) as [BookCount], lb.BranchName, b.Title, ba.AuthorName
from Book as B inner join Book_Authors as BA
	on b.Book_ID = ba.Book_ID
		inner join Book_Loans as BL
	on  b.Book_ID = BL.Book_ID
		inner join Library_Branch as LB
	on BL.BranchID = LB.BranchID
Where Ba.AuthorName like '%' + @Author + '%'
Group By lb.BranchName, b.Title, ba.AuthorName

First example being a specified query for books by a specific author at a specific library. The second is a stored procedure for searching the database based on the Author

Second Reason...

It is a job that ultimately pays out more so than other jobs. There are a few reasons for this.

  • You are consistently learning in this field.
  • The imposter syndrome is all in your head, and everyone has their own pros and cons regarding their education.
  • You can use computer skills to solve everyday problems, or even just small things that you have to repeat. Computers are great at repeating things.

Overall there doesn't seem much of a downside toward learning how to code because...

  1. Even if you are a business man, learning how to do basic database management can help you solidify your job.
  2. The stigma against coding being incredibly difficult is pretty much wrong. There is only advanced math usage in higher forms of programming that most programmers don't even have to worry about.
  3. You get to create cool things! Learning code can allow you to creat cool gadgets or applications that others can use too.
